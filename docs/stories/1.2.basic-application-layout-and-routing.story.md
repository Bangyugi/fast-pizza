# Story 1.2: Basic Application Layout & Routing

## Status

Done

## Story

**As a** Customer,
**I want** to see a consistent application layout with a header and main content area,
**so that** I can easily understand the site structure and navigate my journey.

## Acceptance Criteria

1. React Router is installed and configured to handle client-side navigation.
2. A primary `AppLayout` component is created which renders a header and a main content area for pages.
3. The header prominently displays the company name, "Fast React Pizza Co."
4. A placeholder for a cart summary/link is present in the header.
5. A basic home route (`/`) is configured to render content inside the `AppLayout`.

## Tasks / Subtasks

- [ ] Task 1: Install and configure React Router. (AC: 1)

  - [x] Subtask 1.1: Install `react-router-dom` dependency.
  - [x] Subtask 1.2: Replace the content of `App.jsx` to define the application's routes using `createBrowserRouter`.
  - [x] Subtask 1.3: Create a root layout route that renders the `AppLayout` component.
  - [x] Subtask 1.4: Configure a child route for the home page (`/`) that renders the `Home` component from `src/ui/Home.jsx`.
  - [x] Subtask 1.5: Update `main.jsx` to render the `RouterProvider`.

- [ ] Task 2: Create the main application layout. (AC: 2, 3, 4)
  - [x] Subtask 2.1: Create a new file `src/ui/AppLayout.jsx`.
  - [x] Subtask 2.2: Implement the `AppLayout` component to include a `<header>`, `<main>`, and a placeholder for a cart overview.
  - [x] Subtask 2.3: Use React Router's `<Outlet>` component in the `main` section to render the content of child routes.
  - [x] Subtask 2.4: In the header, add the company name "Fast React Pizza Co." and a link to the home page (`/`).
  - [x] Subtask 2.5: Add a placeholder component `CartOverview.jsx` from `src/features/Cart/CartOverview.jsx` to the layout.

### Component Specifications

- **`AppLayout.jsx`**: A new layout component to be created. It will serve as the main structure for the application, containing the header and the main content area where routed pages will be rendered.
- **`Home.jsx`**: An existing component in `src/ui/` that will be used as the content for the home route (`/`).
- **`CartOverview.jsx`**: An existing placeholder component in `src/features/Cart/` that should be rendered in the layout.

### File Locations

- **`src/App.jsx`**: This file's content must be **replaced** with the routing configuration. [Source: docs/brownfield-architecture/source-tree-and-module-organization.md#key-modules-and-their-purpose]
- **`src/main.jsx`**: This file must be updated to use the React Router provider. [Source: docs/brownfield-architecture/source-tree-and-module-organization.md#key-modules-and-their-purpose]
- **`src/ui/AppLayout.jsx`**: New file to be created. This aligns with the existing structure of placing shared UI components in the `src/ui/` directory. [Source: docs/brownfield-architecture/source-tree-and-module-organization.md#project-structure-actual]

### Technical Constraints

- The application uses React Router for navigation. The existing codebase does not have it, so it must be added.
- The existing `App.jsx` is non-functional boilerplate and its content must be replaced entirely with the router setup.
