# Story 1.4: Individual Pizza Item Display

## Status
Draft

## Story
**As a** Customer,
**I want** to see the details of each pizza, including its name, ingredients, price, and availability,
**so that** I can make an informed choice about my order.

## Acceptance Criteria
1. The `MenuItem` component correctly renders the pizza's image, name, ingredients, and unit price.
2. If a pizza is marked as "sold out," its appearance is visually distinct (e.g., greyed out) and the price is replaced with the text "Sold Out."
3. A non-functional "Add to cart" button is displayed on all available pizza items. This button will have no action in this story.

## Tasks / Subtasks
- [ ] Task 1: Implement `MenuItem` component to display pizza details. (AC: 1)
  - [ ] Subtask 1.1: Modify `src/features/Menu/MenuItem.jsx` to receive `pizza` object as a prop.
  - [ ] Subtask 1.2: Render `pizza.imageUrl`, `pizza.name`, `pizza.ingredients`, and `pizza.unitPrice`.
- [ ] Task 2: Implement "Sold Out" logic and visual styling. (AC: 2)
  - [ ] Subtask 2.1: Add conditional rendering to `MenuItem` to check `pizza.soldOut`.
  - [ ] Subtask 2.2: Apply visual styling (e.g., CSS class) to grey out the item if `soldOut` is true.
  - [ ] Subtask 2.3: Display "Sold Out" text in place of `unitPrice` if `soldOut` is true.
- [ ] Task 3: Add non-functional "Add to cart" button. (AC: 3)
  - [ ] Subtask 3.1: Add a button with text "Add to cart" to `MenuItem`.
  - [ ] Subtask 3.2: Ensure the button is only displayed for available pizzas (i.e., `!pizza.soldOut`).

## Dev Notes
### Previous Story Insights
Story 1.3 was in 'Draft' status when this story was created. This story depends on the data fetching and initial display from 1.3 being functional. The developer should ensure Story 1.3's implementation is stable before proceeding.

### Data Models
The data structure for a pizza item will be implicitly defined by the `Fast React Pizza API` response. It should contain fields like `name`, `ingredients`, `unitPrice`, `imageUrl`, and `soldOut`. [Source: docs/brownfield-architecture/data-models-and-apis.md#data-models]

### API Specifications
Story 1.3 already established fetching the menu. Story 1.4 will consume this already-fetched data. No new API calls are needed for this story. The `getMenu` function in `src/services/apiRestaurant.js` provides the pizza data. [Source: docs/brownfield-architecture/data-models-and-apis.md#api-specifications]

### Component Specifications
*   `MenuItem.jsx`: This component (located in `src/features/Menu/MenuItem.jsx`) will receive a `pizza` object as a prop. It needs to render `pizza.imageUrl`, `pizza.name`, `pizza.ingredients`, `pizza.unitPrice`. It also needs conditional rendering based on `pizza.soldOut` to visually distinguish sold-out items (e.g., apply a CSS class for greying out) and display "Sold Out" instead of the price. A non-functional "Add to cart" button needs to be present for available pizzas. [Source: docs/brownfield-architecture/source-tree-and-module-organization.md#key-modules-and-their-purpose]

### File Locations
*   `src/features/Menu/MenuItem.jsx`: This file will be modified to implement the display logic for individual pizza items. [Source: docs/brownfield-architecture/source-tree-and-module-organization.md#key-modules-and-their-purpose]

### Testing Requirements
Manual visual confirmation is sufficient for this story. Verify that pizza details are displayed correctly (image, name, ingredients, price), sold-out items are visually distinct (greyed out, "Sold Out" text), and the "Add to cart" button appears for available items. [Source: docs/brownfield-architecture/testing-reality.md#current-test-coverage]
